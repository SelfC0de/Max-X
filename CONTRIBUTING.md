# Участие в разработке

## Ветки

- `main` — стабильная версия
- `dev` — активная разработка, PR сюда

## Pull Request

1. `fork` → `feature/твоя-фича` → PR в `dev`
2. Один PR — одна логическая задача
3. Код должен компилироваться: `./gradlew assembleDebug`
4. Без новых сетевых зависимостей за пределами whitelist NetworkGuard
